OOPs Concepts

Object Oriented Programming

Object Oriented Programming (OOPs) is a programming style or paradigm or methodology based on the concept of OBJECTS, Purpose of OOP is to increase the flexibility, improve the program maintainability, code reusability and provide easy solution for large and complex problem,

Concepts

  • Object
  • Class
  • Encapsulation
  • Polymorphism
  • Inheritence
  • Abstraction

OOPS concepts are very important and without this concepts, we’ll not be able to design proper system in the object-oriented programming model.

inheritance

Inheritance is process of acquiring features, where one class will be allowed to inherit the properties and behavior of another class.
Example: Build a new class with extending the feature of other class, this provide code reusability, and new class can reuse the extended class methods and properties, and same time new class can add a new methods and properties.
The class extends the feature of another class is called sub class or Child class or Derived class.
The class whose feature and properties inherited is called super class or Parent class or Base class.
To inherit from a class, use the extends keyword.

Polymorphism

Polymorphism is ability of performing single action in many forms, there’re two type of polymorphism.
  • Compile Time polymorphism or Static binding
  • Run Time polymorphism or Dynamic Binding

It is known as static polymorphism. This type of polymorphism can be achieved by using method overloading.

Abstraction

Abstraction - Process of hiding the implementation details and showing only functionality (relevant details) to the user, it helps to reduce programming complexity and effort. Abstraction is accomplished using Abstract classes and interfaces

Abstract Class

A class which is declared with abstract called as an abstract class, it can contain abstract methods and concrete methods
Normal class can’t have abstract methods.

Abstract Method

A method which is declared with abstract and only method declaration / signature and no implementation, it called as an abstract method. all the abstract method must implement the derived class. .
In java, abstraction is achieved by interfaces and abstract classes. We can achieve 100% abstraction using interfaces. . 
public abstract class AbstractList extends AbstractCollection implements List{

Interface

An interface is a blueprint of a class, and bringing the standard across the implementation class, Interface also like a class, an interface can have methods and variable but only declaration / signature, methods are by default abstract, it’s used to achieve the abstraction and interface can support the multiple inheritance
Use Interface keyword
Since Java 8, we can have default and static methods in an interface.
Since Java 9, we can have private methods in an interface.
